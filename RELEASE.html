GXemul:  Release notes

Release notes for GXemul 0.4.6.6

Copyright (C) 2003-2008 Anders Gavare


GXemul is a framework for full-system computer architecture emulation. Several processor architectures and machine types have been implemented. It is working well enough to allow unmodified "guest" operating systems to run inside the emulator, as if they were running on real hardware.

The emulator emulates (networks of) real machines. The machines may consist of ARM, MIPS, PowerPC, and SuperH processors, and various surrounding hardware components such as framebuffers, busses, interrupt controllers, ethernet controllers, disk controllers, and serial port controllers.

The documentation lists the machines and guest operating systems that can be regarded as "working" in GXemul. The best working guest operating systems are NetBSD/pmax and NetBSD/cats.


The changes between release 0.4.6 and 0.4.6.1 are mostly focused on documentation updates, to reflect new releases of guest operating systems:

  • OpenBSD 4.2 has been released. OpenBSD/landisk 4.2 works in GXemul.

  • NetBSD 4.0 has been released. The installation instructions in GXemul's documentation have been updated to reflect this.

    • In most cases this simply means that NetBSD 4.0 should work, in addition to NetBSD 3.1.

    Some specifics worth mentioning are:

    • For NetBSD/pmax, the change from 3.1 to 4.0 means that X Windows is now working again out-of-the-box. (Simply run startx as root to start X.)
    • For NetBSD/arc, the supported version has jumped from 1.6.2 to 4.0.
    • NetBSD/landisk has been released, and works in GXemul.
    • NetBSD/prep is unfortunately still at NetBSD 2.1.
    • In some emulation modes, such as evbmips, hpcmips, and perhaps some more, NetBSD 4.0 uses the clock in a way which is not yet implemented in the emulator. (NetBSD 4.0 works, but NetBSD 3.1 worked better.)
    • (I have not tried some of the "heavier" emulation setups, such as NetBSD/sgimips.)

The changes between release 0.4.6.1 and 0.4.6.3 are:

  • Applying a patch from Oleksandr Tymoshenko for the wdc device, fixing the order of 16-bit words within 32-bit words in the identify struct, and the error register is not cleared on reads any more.
  • The disk device used in the test machines can now access disk images larger than 4 GBs when emulating 32-bit architectures (as suggested by Julian Elischer).

The changes between release 0.4.6.3 and 0.4.6.5 are:

  • For ARM emulation, a special case for PC-relative load which wasn't implemented before has now been implemented. (The problem could be triggered by attempting large builds using NetBSD's pkgsrc inside e.g. NetBSD/cats.)
  • The ARM 'bkpt' instruction has been implemented (patch from Jiri Svoboda).

The changes between release 0.4.6.5 and 0.4.6.6 are:

  • Applying a patch from Kiyotaka Inaba which causes an ARM UND exception to be generated by the instruction word 0xe6000011, which is needed by gdb when running inside the emulator.
  • Fix for a free() bug in bootblock_iso9660.c.
  • OpenBSD/landisk installation instructions have been updated to 4.4.
  • Most NetBSD installation instructions have been updated to 4.0.1.

To build the emulator, run the configure script, and then run make. This should work on most Unix-like systems, with few or no modifications to the source code.
